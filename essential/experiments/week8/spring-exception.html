Spring - Exception Handling

This Web Page is mainly about the exception handling in Spring

Exceptions can be handled using the ExceptionHandler. The state can be maintained and a proper error response can be sent back to the client incase of error conditions

                       @ExceptionHandler(EmotionNotFoundException.class)
                        @ResponseStatus(HttpStatus.NOT_FOUND)
                             public @ResponseBody Error emotionNotFound(EmotionNotFoundException e) {
                            String emotion = e.getEmotion();
                            return new Error(4, "Response [" + emotion + "] not found");
                        }

The error object has the errorId and the message to be populated

                     public class EmotionNotFoundException extends RuntimeException {
                         private String emotion;
    
                         public EmotionNotFoundException(String emotion) {
                            this.emotion = emotion;
                        }
    
                         public String getEmotion() {
                            return emotion;
                        }

                    }

I am using Spring's ExceptionHandler to handle bad requests. When a requestById is made for an id that has no data in database, a proper error response has to be maintanied to be displayed to the browser
